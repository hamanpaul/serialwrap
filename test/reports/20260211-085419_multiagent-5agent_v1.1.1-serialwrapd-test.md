## e2e 驗證條件（test_multiagent_e2e）
- fake UART 背景持續噴 `KDBG:tick` 雜訊
- 5 個 agent、3 輪，共 15 筆命令並發提交
- 命令樣本包含：`ifconfig`, `wifi restart`, `iw dev wlan0 scan`, `iw dev wlan0 link`, `logread -f`
- 驗證：
  - submit 成功 15/15
  - `cmd_id` 唯一 15/15
  - WAL 中 agent TX 記錄 15/15 且每 `cmd_id` 僅 1 筆
  - RX 可找到每筆 `RESULT:<cmd>:OK`
  - 每 agent 恰有 3 筆提交可追溯

## 結論
- 多 agent 同 UART access 在 broker 模式下可穩定分時序列化。
- 在持續雜訊下，RAW WAL 仍可完整回收並回溯到各 agent 指令。
